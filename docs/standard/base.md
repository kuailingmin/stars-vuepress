# 前端规范


## 核心价值

1. 提升可维护性
   - 统一代码风格（锁进、命名、注释）降低后期维护成本
   - 语义化HTML标签（如 header、article）增强代码可读性

2. 优化团队协作
   - 减少风格争议，聚焦业务逻辑开发
   - 规范的文件结构（如 src/components/）便于快速定位代码

3. 保障性能和安全
   - 资源压缩、合并请求（如CSS/JS文件≤10个）提升加载速度
   - 防御XSS攻击（转义敏感字符如 <>&）


## 核心价值规范分类

1. 基础规范
   - 比如html、css、js、ts等语法的规范

2. 组件与框架规范
   - vue组件规范制定
   - 路由、状态管理等规范制定

3. 工程化与性能规范
   - 资源优化： 
      - 图片：合并雪碧图、使用字体图标、首屏外图片懒加载
      - 静态资源：开启Gzip压缩、CDN加速

   - 安全与兼容性：
      - 外部链接加 rel="noopener" 防安全漏洞
      - 浏览器分级支持（如完全兼容Chrome/Firefox/Edge）

## 总结

前端规范的核心是 一致性、可维护性、性能。制定时需结合团队技术栈和业务场景，通过工具约束（ESLint）和文档沉淀（如README规范）确保落地。定期复审规范以适应技术演进（如CSS新特性）
